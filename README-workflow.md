# Claude Dev Workflow Setup

## Folder Structure

```
your-angular-project/
├── .claude/
│   ├── CLAUDE.md          ← Workflow instructions (Claude reads this automatically)
│   ├── settings.json      ← Hook config + permissions
│   ├── hooks/
│   │   └── pre-prompt.sh  ← Auto-runs when you use JIRA_STORIES: prompt
│   └── jira-context.md    ← Auto-generated by hook (don't edit manually)
```

---

## One-Time Setup

1. **Copy** the `.claude/` folder into your Angular project root.

2. **Make the hook executable:**
   ```bash
   chmod +x .claude/hooks/pre-prompt.sh
   ```

3. **Verify your jira-cli path** in `hooks/pre-prompt.sh`:
   ```bash
   JIRA_CLI="C:/Users/suryathota/tools/jira-cli"
   ```
   Adjust the `jira-cli issue view` command syntax to match your `jira-cli_SPEC.md.txt`.

4. **Verify Claude Code picks up project config** (not global):
   ```bash
   # Run this from your project root
   claude --show-config
   # Should show .claude/settings.json from your project
   ```

---

## How to Use

Simply prompt Claude Code like this:

```
JIRA_STORIES: PROJ-123, PROJ-124
DESCRIPTION: Add a new user profile page with edit functionality.
             The form should validate email and phone fields.
MOCKUP_FOLDER: C:\Users\suryathota\mockups\profile-screens
```

**Without mockups:**
```
JIRA_STORIES: PROJ-125
DESCRIPTION: Fix the pagination bug on the orders list page.
             Page resets to 1 when filters are applied — should maintain current page.
```

## What Happens Automatically

| Step | Who does it | What |
|------|-------------|------|
| 1 | Pre-hook | Detects `JIRA_STORIES:`, fetches all stories via jira-cli |
| 2 | Pre-hook | Writes `.claude/jira-context.md` with story details + branch name |
| 3 | Claude | Reads context, checks out branch from develop via bitbucket-cli |
| 4 | Claude | Reviews mockups (if provided), plans + implements changes |
| 5 | Claude | Runs `ng build`, fixes any errors |
| 6 | Claude | Shows you a summary, **waits for your approval** |
| 7 | Claude | On your YES → commits + raises PR to develop |

## Fix: Project Config vs Global Config

Claude Code loads configs in this order (both are loaded, project overrides global):
- Global: `~/.claude/settings.json` and `~/.claude/CLAUDE.md`
- Project: `.claude/settings.json` and `.claude/CLAUDE.md`

**Important:** Always launch Claude Code **from your project root**:
```bash
cd C:\path\to\your-angular-project
claude
```

If Claude still seems to ignore project config, check:
```bash
# Confirm .claude folder is at the root
ls -la .claude/

# Confirm settings.json is valid JSON
cat .claude/settings.json
```

## Customization

- **Change build command:** Edit `CLAUDE.md` → replace `ng build` with `ng build --configuration=production`
- **Add linting step:** Add `ng lint` before `ng build` in CLAUDE.md Step 5
- **Change branch prefix:** Edit `pre-prompt.sh` → change `feature/` to `feat/`
- **Add default reviewers to PR:** Update the bitbucket-cli command in CLAUDE.md Step 7
